DOO - Canevas du TP 3 : Encapsulation et utilisation de type

Attendus :
- Développer une classe utilisant une autre classe;
- Refondre une classe encapsulée;
A rendre en fin de TP :
- Le compte-rendu (PDF + DOC/ODT),
- les sources (y-compris Angle.java et les tests), et la documentation complète des classes Pixel et Complexe dans une archive ZIP (seul format accepté).
- Rien d'autre (pas les fichiers .class, ni l'énoncé).
Evaluation :

L'évaluation de ce TP portera sur la qualité du développement et sur la forme du rendu (fichier source qui compile et exécute, compte-rendu sérieux, documentation complète, absence de fichiers parasites) plutôt que sur la quantité de code produite. Le résultat sera un bonus/malus adossé à la note du prochain TP noté.

Partie 1 : Type Complexe

En s'appuyant sur le type Angle du TP précédent, écrire, tester et documenter (javadoc) un type Complexe représentant un nombre complexe c par sa partie réelle a et sa partie imaginaire b. Notation : c = a + i.b

  • Constructeurs :
    • - un constructeur principal à partir de deux valeurs réelles pour les parties réelle et imaginaire (prévoir à la suite le toString pour pouvoir tester),
    - un constructeur à partir d'un module et d'un argument,
    - un constructeur par recopie.
  • Accesseurs :
    • - une méthode de conversion en chaîne de caractères,
    - une méthode partieReelle qui retourne la partie réelle,
    - une méthode partieImaginaire qui retourne la partie imaginaire,
    - une méthode module qui retourne le module,
    - une méthode argument qui retourne l'argument,
    - une méthode estReelPur qui teste si le complexe est réel pur (à ε près),
    - une méthode conjugue qui retourne le complexe conjugué : a -i.b,
    - une méthode somme qui retourne la somme de deux complexes : (a1 + a2) + i.(b1 + b2),
    - une méthode produit qui retourne le produit avec un autre complexe : (a1.a2 - b1.b2) + i (a1.b2 + a2.b1),
  • Mutateurs :
    • - une méthode ajouter qui ajoute la valeur d'un autre complexe,
    - une méthode multiplier qui multiplie par un autre complexe,

    Partie 2 : Refonte du type Complexe

    Recopier tous les fichiers sources .java (y compris les tests) dans un nouveau répertoire v2, puis travailler dans ce répertoire v2.

    Réécrire la classe Complexe.java en prenant cette fois comme attributs un module et un argument. En principe, il suffit de repartir de l'ancien fichier Java, de changer les attributs et de rectifier le contenu des méthodes. Les tests réalisés précédemment doivent encore fonctionner.

    Compléter cette classe avec :
    - une méthode inverse qui retourne l'inverse du complexe : (a - i.b) / (a.a + b.b),
    - une méthode inverser qui remplace le complexe par son inverse.
